# News RAG System

Technical assignment project for a RAG-based news processing system.

The system is intended to:
- fetch news from configurable external APIs
- normalize and enrich articles with sentiment and embeddings
- store them in OpenSearch
- expose a REST API for semantic search

Current repository structure:
- `news-ingestion-service`: article ingestion and enrichment pipeline
- `search-service`: semantic search REST API
- `config/sources.yaml`: external news source configuration
- `docker-compose.yml`: local OpenSearch setup

## Local OpenSearch

Start OpenSearch:

```bash
docker compose up -d opensearch
```

Check that it is running:

```bash
curl http://localhost:9200
```

Create the local news article index:

```powershell
.\scripts\opensearch\create-news-article-index.ps1
```

The script creates an index named `news_article` with k-NN enabled and a 16-dimensional `embedding` vector, matching the deterministic embeddings generated by the ingestion service. It uses one primary shard and zero replicas so the index can be green in the local single-node OpenSearch setup. If the index already exists, the script prints a message and exits successfully.

To delete and recreate the local index:

```powershell
.\scripts\opensearch\create-news-article-index.ps1 -Recreate
```

Verify the index exists:

```bash
curl http://localhost:9200/news_article
```

Stop OpenSearch:

```bash
docker compose down
```

OpenSearch data is stored in the `opensearch-data` Docker volume so local indexed data can survive container restarts.
